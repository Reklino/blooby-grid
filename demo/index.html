Ghost Grid

Summon your grid from the depths of css hell so that you might visualize and conquer it once and for all.

Columns

Visualize your columns while you develop so you can rest assured you're within your bounds and, more importantly, make design decisions based on something you can see.

Baseline Grid

Use Ghost Grid's basline grid to ensure your site/app maintains a consistent vertical rhythm.

Multiple Breakpoints

Use Ghost Grid to visualize a responsive column layout by setting up different columns at various breakpoints. Resize this screen to see an example.

Fluid, Static, or Flex

Whether you prefer fluid grids (percentage based) or static grids (static value based), Ghost Grid can math it.

Why did I make this?

I wanted it! It's super handy to keep yourself in check whilst developing the structure of an application or website.

More and more, I find myself designing with code, and I don't intend to stop. It's an amazing feeling when you're able to craft a desisn by typing up some html and css, especially when you can do it quickly. Some things, however, are lost in this design process. One of which is visual guides, grids, and layout tools that you find in design & prototyping applications.

There have been other attempts at bridging this gap which I have taken heavy inspiration from. Most notably is the Golden Grid System and its 'Golden Gridlet' overlay. My goal with Ghost Grid was to simply create an easily customizable, dependency free version of what Joni Korpi did with GGS that had a few more features.

Define the container

If the container's max width is undefined, it will be set to the full width of the the page. The position defaults to center but accepts 'left', 'center', or 'right'; 


bloobyGrid.container = 960,
bloobyGrid.position = 'center';

Add a breakpoint or two

Each breakpoint object has 4 required properties:

  • The 'point' property accepts any valid css media query.
  • The 'columns' property should be set to however many columns are active at this breakpoint.
  • The 'gutters' determine the space between the columns in percentage.
  • The 'baseLineHeight' sets the base lineheight in pixels.


bloobyGrid.breaks = [
    {
        point : "(min-width: 800px)",
        columns : 12,
        gutters : .33,
        baseLineHeight : 24
    },
    {
        point : "(min-width: 300px)",
        columns : 3,
        gutters : .33,
        baseLineHeight : 24
    }
];

Initiate!

When you've configured it, just run bloobyGrid.init();. To the right is what your entire setup might look like. 


<!DOCTYPE html>
<html>
<head>
	<link rel="stylesheet" type="text/css" href="blooby-grid.css">
</head>
<body>
	<script src="blooby-grid.js"></script>
	<script>
		bloobyGrid.baseLineHeight    = 24,
		bloobyGrid.container         = 960,
		bloobyGrid.containerPosition = 'center',
		bloobyGrid.breaks = [
		    {
		        point : "(min-width: 800px)",
		        columns : 12,
		        gutters : .33
		    }
		];
		bloobyGrid.gridInit();
	</script>
</body>
</html>

Plays well with Susy

I love using Susy to quickly get started with complex fluid grid layouts so I made sure this worked well with it. The structure of this project was inspired heavily by Susy and should therefore be familiar to any past users of Susy. 


bloobyGrid.container = 960,
bloobyGrid.breaks = [
    {
        point : "(min-width: 800px)",
        columns : 12,
        gutters : .33
    }
];
			

$susy: (
    columns: 12,
    gutters: .45,
    gutter-position: inside,
    container: 960px
);
